The HTML Boilerplate

Introduction to the HTML Boilerplate

Now that we have seen how HTML elements and tags work, it is time to explore the bigger picture, which is called the HTML Boilerplate. This will allow us to understand the structure of our HTML files, just like how there is structure to letters that you would write.

For example, in a letter, there is a section where you have your address, then you probably include a greeting such as "Dear so and so," followed by the body of your letter, and finally, you would write something like "Yours sincerely" and then your name and signature.

Similarly, there is a structure to our HTML files, and it looks something like this. This is called a Boilerplate. It is what you usually start off with when you create any new HTML file. We will go through it line by line so that we understand it.

The Doctype Declaration

At the top of every HTML file, there should be a "Doctype declaration." This tag tells the browser which version of HTML the file was written in.

The latest version of HTML is HTML5, and the Doctype declaration looks like this:

<!DOCTYPE html>

This line tells any browser reading our HTML file that we have written our code in HTML Version 5.

The Root HTML Element

The next part is the actual <html> element. This is the root of the document, meaning that everything else—every other element, whether it is an h1, an image, or a link—goes inside the opening and closing tags of the html element.

This element has a lang attribute, which specifies the language of the text content inside that element. For example, if you are writing a website targeted at English speakers, you would set lang="en".

While this attribute is not very important for users who can see your website, it is more important for users who cannot. Various screen readers and assistive technologies use this language attribute to pronounce the words correctly when reading out the website.

The Hamburger Analogy for HTML Structure

With the html element, we start to see the beginnings of a hamburger analogy: there is a bun at the top and a bun at the bottom. Everything that belongs inside the html element is sandwiched between these two buns.

This analogy helps visualize the nesting structure of HTML elements.

The Head Element

The next part of the HTML boilerplate is the <head> element. This section contains important information about our website that is not displayed to the user.

It includes things that help the website render correctly in the browser but does not include any content like text or images that the user will see.

One essential tag in the head section is the meta tag for character set encoding, usually set as UTF-8. This ensures that the characters used on your website display correctly, including emojis and special symbols like multiplication and division signs.

By convention, this tag is included as is in the HTML boilerplate using the UTF-8 character set, which is one of the most common and widely supported.

The Title Element

Another important tag in the head section is the <title> element. The content inside the opening and closing <title> tags specifies the title of your website, which is displayed in the browser's tab bar.

For example, if you right-click on any website in Chrome and select "View Page Source," you will see the DOCTYPE, the html element, the head section with the character set meta tag, and the title tag specifying the page title.

In our example, the title will simply say "My Website."

Additional Head Elements

There are many other tags that can go into the head element, as seen on websites like Google's homepage. However, for now, the character set meta tag and the title are the most important elements that will always go into your HTML files.

We will cover other tags as we come across them.

The Body Element

The next part of the HTML boilerplate is the <body> element. Just like a person, after the head element comes the body element.

Inside the body element is where we will spend the majority of our time creating and writing our website. This is where all the content of the website goes: text, titles, images, links, and everything else you can do with HTML to create content and structure.

All of this content goes between the opening and closing tags of the body element.

Content Inside the Body

Inside the body, you can include headings such as h1, images, links, and many other HTML elements. This is essentially where the "meat" of your website lives.

Once the HTML document is complete, the structure looks like a hamburger: the html element is the bun, the head and body elements are the layers inside, and the content is sandwiched accordingly.

By default, most websites start with a boilerplate that looks like this.

Writing Your Own Boilerplate

When starting out, it is often recommended to type out this boilerplate manually when creating new websites. You can practice this now by going into VS Code, typing out these elements, and creating your website entirely from scratch.

This practice helps you understand the structure and nesting of HTML elements.

Nesting and Indentation in HTML

Nesting is a fundamental concept in HTML and programming in general. Using the hamburger analogy, the html element is the top and bottom bun, and everything inside is sandwiched between.

Similarly, the body element has opening and closing tags, and the content inside is like the filling between slices of tomato.

Some HTML tags have opening and closing tags on the same line if the content is short, but they still have a beginning and an end.

Proper indentation is important when writing code. Indenting nested elements helps visualize the structure clearly. For example, the head section is indented inside the html section, just like lettuce inside a hamburger.

This keeps the code tidy and easy to read at a glance.

VS Code Shortcut for Boilerplate

Using VS Code, there is a shortcut to avoid typing the boilerplate every time. Once you have typed it out once or twice and know how it looks, you can save time by typing an exclamation mark (!) and pressing Enter on the first suggestion.

This automatically inserts the entire boilerplate code for you, saving time when creating many HTML documents.

Note that this only works if you have created a file with the .html extension. If you save the file with another extension, this shortcut will not work.

Additional Meta Tags

There are a couple of meta tags you might see in the boilerplate that we haven't covered yet.

One is <meta http-equiv="X-UA-Compatible" content="IE=edge">, which keeps the code compatible with Internet Explorer. However, Internet Explorer has been deprecated and is no longer in use, so you can delete this line from your VS Code autogenerated boilerplate to keep your code clean.

Another important meta tag is the viewport tag, which defines how the website should be displayed relative to the screen it is rendered on. This tag tells the browser how to display your website when it first opens.

Key Takeaways

  • The HTML Boilerplate provides the essential structure for every HTML document.
  • The <!DOCTYPE html> declaration specifies the HTML version, which is HTML5.
  • The <html> element is the root, containing all other elements, with a lang attribute for language.
  • The <head> section contains metadata like character encoding and the page title, which are not displayed to users.
  • The <body> section contains all the visible content of the webpage.
  • Proper nesting and indentation in HTML help maintain clear and organized code.
  • VS Code offers a shortcut by typing ! and pressing Enter to generate the HTML boilerplate automatically.
  • Some meta tags, like those for Internet Explorer compatibility, are now deprecated and can be omitted for cleaner code.
